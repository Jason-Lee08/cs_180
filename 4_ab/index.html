Image Rectification

In this section, I will demonstrate how we can rectify (or, as I think of it, rectangular-ify) an image using homographies.

Suppose I have this image of a circuit on a bread board that is taken at an angle, so it’s difficult to see the exact locations of the wires. First, I gather the 4 corresponding points by selecting all the corners of the breadboard (which are not rectangular in the original image) and define another 4 points as my target rectangle (see the red rectangle on the right image).

Warping:

First, I solve for H by using the 4 corresponding points gathered above, such that H will warp any of the breadboard corners to become a corner of the rectangle. Then, I take the four corners of the original image and warp them according to H to get the destination boundary. From this destination boundary, I am able to achieve an x_offset and y_offset by taking the minimum x/y values of the four corners.

Interpolation:

Finally, I implement interpolation to retrieve the color values of the warped image. To do this, I create a meshgrid using the corners of the warped corners and apply H’s inverse to get back a point in the original image to interpolate on. I utilize SciPy’s RegularGridInterpolator function with the ‘linear’ method to handle the interpolation. To clean up, I mask out all of the nan entries from the final image by making them transparent.

Stitching/Blending

Here’s the stitched combination of the left/center images:

To stitch the images together, I utilize the offsets I found using the warping function above. These offsets are the minimum x/y values of the corners transformed using H of the warp-image. They determine where to place the transformed image relative to the non-transformed image when we stitch them together. For example, in the example shown above, x_offset and y_offset are both negative since the transformed image is to the left and above the non-transformed image.

To blend the images together, I use very simple average-blending. To do this, I create a blank image (np.zeros(…)) and calculate a mask of the overlap of the two images. Then, in the blank image, I add the average of the two images in the overlapping area. Afterward, I add in the remaining non-averaged, non-overlapping sections of both images. Finally, I compute an AND mask of the two images and set the alpha (transparency value) within the mask to be 1 (visible).

In the image above, I tried implementing a Laplacian stack which does an excellent job of removing the artifacts produced by image boundaries. However, running the Laplacian blending was very slow, so it was very difficult to tune the parameters to make the blending more smooth.

While average-blending left artifacts of the image boundaries in the final mosaic, I felt like for how fast/simple of a solution it is, it did a very good job in blending the images. In the examples shown below, I use the faster average-blending due to time constraints.

Project 4B - Feature Matching for Auto-Stitching

In part B, I implement automatic image stitching following the ideas proposed in “Multi-Image Matching using Multi-Scale Oriented Patches” by Brown et al. There are 5 steps to follow to implement automatic image-stitching:

  1. Harris Interest Point Detector
  1. Adaptive Non-Maximal Suppression
  1. Feature Descriptor extraction
  1. Feature Matching
  1. RANSAC (Random sample consensus)

  1. Harris Interest Point-Detector

At a high level, the Harris Interest Point Detector uses the sum of products of gradients around a region in an image to determine whether a point in an image is a corner or not.

  1. Adaptive Non-Maximal Suppression (ANMS)

Basically, ANMS helps reduce the abundance of points from the Harris Point Detector down to a finer set of “best” points. The idea behind ANMS is to assign a score to each point based on how good it is, then keep track of its suppression radius - the distance to the point with a higher score. If this suppression radius is high enough, the point is kept. Using this point filtering mechanism, only the best points (with high scores) that are spread out are kept.

My implementation stores all the previously processed corners in order of response strength. Then, I search over each point and determine where the closest point with the same strength response is located, and store this in another array called radii. Then, I sort the corner points based on its radii in descending order and select the top num_features points.

  1. Feature Descriptor extraction

This feature extraction tool basically iterates over each significant point (the points remaining after applying ANMS) and takes a 40x40 window surrounding the significant point, then picks every 5th point in the window, resulting in an 8x8 feature. My implementation is very straightforward and exactly what I said above.

Examples of these patches can be seen in the next section.

  1. Feature Matching

To match these features/points between images, I simply find the closest feature in the other image and add it based on whether the 1-NN error / 2-NN error is below a certain threshold. To start, I iterate over each point in img1 and subtract it from every other feature in img2, then compute the L2 norm of the 8x8 difference matrix. Then, I sort it in ascending order and take the 2 closest features to compare their error ratios. If the 1-NN error / 2-NN error is below Lowe’s value, or in my case, 0.4, then it’s added as a corresponding point. Then, I acquire the corresponding points by utilizing the fact that img1's feature index corresponds to img2's feature index, thus these same indices correspond for their points.

Note that sometimes the corresponding points will point to completely non-corresponding areas in each image due to chance.

  1. RANSAC

RANSAC takes in many potentially correlated points and finds the 4 best points to compute a homography by maximizing the number of inliers (after applying the homography to pts1, # of pts within a certain threshold of pts2). To do this I use np.rand to generate random indices (without replacement) and run 1000 iterations of random sampling to try to find the best points.

After finding the Homography best_H that produces the maximum number of inliers, I warp the target image and stitch them together using the functions I wrote in project 4A above.
